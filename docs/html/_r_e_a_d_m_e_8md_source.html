6 This is a generic implementation of `then` for C++14, allowing you to chain together futures, promises and so forth easily and efficiently.

8 ## Example:
9 ```C++
10 #include "thenable.hpp"
11 
12 #include <iostream>
13 #include <chrono>
14 #include <thread>
15 #include <assert.h>
16 
17 using namespace std;
18 using namespace thenable;
19 
20 int main() {
21  ThenablePromise<int> p, k;
22 
23  cout << "Starting asynchronous tasks on thread: " << this_thread::get_id() << endl;
24 
25  p.then( [&k]( int i ) {
26  assert( i == 10 );
27 
28  cout << "Hello, World! from thread: " << this_thread::get_id() << endl;
29 
30  k.set_value( 421 );
31 
32  return k.get_future();
33 
34  } ).then( []( int i ) {
35  assert( i = 421 );
36 
37  cout << "Hello, again! from thread: " << this_thread::get_id() << endl;
38 
39  }, then_launch::detached );
40 
41  //Launch a new thread to resolve the original promise.
42  thread( [&p] {
43  cout << "Resolving promise on thread: " << this_thread::get_id() << endl;
44 
45  p.set_value( 10 );
46  } ).join();
47 
48  //Just to keep the program alive long enough for the promises/futures to propagate.
49  this_thread::sleep_for( 10s );
50 
51  return 0;
52 }
53 
54 ```
55 
56 The output of the above will be something like:
57 ```
58 Starting asynchronous tasks on thread: 1
59 Resolving promise on thread: 4
60 Hello, World! from thread: 2
61 Hello, again! from thread: 3
62 ```
63 
64 If `THENABLE_DEFAULT_POLICY` is set to `std::launch::deferred`, then the last two actions will be performed on the same thread.
